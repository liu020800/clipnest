# v1.1 Screen OCR Design

## Goal

Provide a stable screen-region OCR workflow:

```text
Ctrl+Shift+O -> transparent full-screen selector -> capture selected region -> OCR -> open save window with recognized text
```

This replaces the earlier clipboard-image OCR design. The app no longer reads image data from the system clipboard for OCR, so normal clipboard text capture stays isolated and reliable.

## User Flow

1. User presses `Ctrl+Shift+O`.
2. Tauri shows the `screen_ocr` window.
3. The window is transparent and always on top.
4. User drags a rectangle over visible desktop content.
5. Frontend calls `capture_screen_ocr_region`.
6. Backend hides the overlay, waits briefly, captures only the selected area, and writes a temporary PNG under app cache.
7. OCR runs on the PNG.
8. Recognized text is stored as pending capture text.
9. The normal save window opens with OCR text prefilled.

## OCR Strategy

Primary engine:

- RapidOCR through Python script.
- Script is bundled with the app.
- It calls the user's local Python environment and RapidOCR/ONNXRuntime packages.
- The script performs small-text preprocessing, confidence filtering, mojibake repair, and narrow text cleanup.
- Output is written as UTF-8 bytes, not via locale-dependent `print`.

Fallback engine:

- WeChatOCR host (`wcocr.exe`) bundled under `resources/ocr_host`.
- Includes `WeChatOcr.dll`, `WeChatOCR.exe`, and `wco_data\Model`.
- It does not call the user's running WeChat client and does not require a logged-in WeChat account.

## Packaging Notes

`src-tauri/build.rs` copies the C# OCR host output from:

```text
scripts/ocr_host/bin/Release/net8.0
```

to:

```text
src-tauri/resources/ocr_host
```

`tauri.conf.json` must include OCR resources recursively:

```json
"resources": [
  "resources/ocr_host/**/*",
  "resources/rapidocr/*"
]
```

Without recursive inclusion, `wco_data\Model` is not installed and WeChatOCR fallback cannot start.

## Verification

Required checks before release:

```bash
dotnet build scripts/ocr_host/ClipNest.OcrHost.csproj -c Release --no-restore
npm run build
cargo test --manifest-path src-tauri/Cargo.toml
npm run tauri -- build
```

After installation, verify:

```powershell
python $env:LOCALAPPDATA\ClipNest\resources\rapidocr\rcr_ocr.py <crop.png> --json
Push-Location $env:LOCALAPPDATA\ClipNest\resources\ocr_host
.\wcocr.exe <crop.png> --json
Pop-Location
```
